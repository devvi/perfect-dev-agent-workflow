## 维护规则

- **谁维护：** Review agent 在 post-merge 阶段写入
- **触发时机：** Implement PR merge 后，自动提取 DESIGN doc 中的设计决策
- **格式要求：** 每篇单独文件，通过 INDEX.md 链接索引
- **不重复的内容：** Issue 级的实现细节（代码 diff、测试用例）不进入 GDD，它们留在 PRD/DESIGN 中
- **GDD 只收：** 系统设计意图、数据结构、常量定义、行为规则——那些下次做新功能时需要查阅的知识
