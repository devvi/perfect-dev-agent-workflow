## 7.2 标题画面

暗色半透明遮罩，显示：
- 游戏标题 + 副标题
- 操作说明（方向键移动、Z 射击、X 交互、SHIFT 暂停）
- 交互式菜单（START GAME / ABOUT），当前支持 ENTER 或 SPACE 选择

ABOUT 页显示当前构建的 commit hash、message 和 date。

## 7.3 HUD

- 右上角：半透明小地图（50% alpha）
- 左上角：得分（score）
- 中央状态消息：门的提示（"NEEDS KEY" / "NEEDS LENGTH N+"）

## 7.4 覆盖层

### 暂停画面（Pause）
半透明遮罩 + "PAUSED" 文字，按 Shift 恢复。

### 游戏结束画面（Game Over）
暗色遮罩 + 统计信息：最终得分、蛇长、探索房间数、杀敌数。

### 胜利画面（Victory）
金色 "DUNGEON CLEARED" + 统计信息。按 Space/Enter 重新开始。

### BOSS 介绍
进入 BOSS 房间时，游戏暂停并显示 BOSS 名称和对话框文字。
按 Space/Enter/方向键开始战斗。

### 版本号标签（Issue #175）
标题画面右下角显示版本号 `"v1.0.0"`（硬编码），便于调试时确认运行版本。

- **位置：** `(CANVAS_SIZE - 10, CANVAS_SIZE - 10)`，`textAlign = 'right'`，距右下角 10px
- **样式：** `rgba(255, 255, 255, 0.3)`，`10px monospace`，半透明不干扰菜单
- **实现：** `renderTitleScreen()` 末尾通过 `ctx.save/restore` 包裹，不污染上下文状态
- **未来：** 版本号当前为硬编码，后续 Issue 可改为动态（从构建环境注入）

## 7.5 渲染尺寸

```javascript
CANVAS_SIZE = 400           // 画布 400×400 像素
CELL_SIZE = 20              // 每格 20 像素（标准房间）
BOSS_CELL_SIZE = 5          // BOSS 房间每格 5 像素（80×80 映射到 400px）
MINIMAP_SIZE = 100          // 小地图 100×100 像素
MINIMAP_MARGIN = 10
```
