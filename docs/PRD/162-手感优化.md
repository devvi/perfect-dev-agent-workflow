# PRD: 手感优化 — 方向键转蛇头输入延迟

> **Issue:** #162 | **Depth:** light | **Type:** enhancement

## 1. Problem

### 当前行为

玩家按下方向键（↑↓←→）后，蛇头**没有即刻响应**，存在明显的视觉延迟，有时延迟较大。输入被正常记录（`nextDirection` 立即更新），但蛇头视觉上需要等到下一个 tick 才发生变化。

### 预期行为

按下方向键后，蛇头应**立刻在视觉上朝新方向转动**（眼睛位置变化），即使实际移动要等到下一个 tick。玩家需要立即确认输入已被接收。

### 用户场景

- 高速移动时（短蛇，tick 间隔 150ms），玩家连按方向键，无法判断输入是否被记录，导致连续无效按键
- 低速移动时（长蛇，tick 间隔可达 800ms），按下一个方向后需要等待最长达 800ms 才能看到蛇头转向，体验极差
- 在狭窄通道中需要精确转向时，延迟导致撞墙

### 根源分析

经过代码勘探，**根本原因不在输入记录环节**（`changeDirection` 是同步的，`nextDirection` 被立即更新），而在于：

1. **渲染层只读 `state.direction`**：`room.js` 的 `drawSnake()`（第 242 行）绘制蛇头眼睛时使用 `state.direction`，而不是 `state.nextDirection`。蛇头直到下一个 tick（`direction = nextDirection`）才会改变朝向。

2. **方向变更后没有立即重绘**：`gameboy.html` 的 `keydown` 处理器（第 539-541 行）调用 `changeDirection` 后没有触发 `render()`，即使渲染逻辑改了，也要等到下一个 tick 的渲染才会显示。

3. **Tick 间隔放大感知延迟**：`BASE_TICK_INTERVAL = 150ms` 到 `MAX_TICK_INTERVAL = 800ms` 的区间意味着玩家可能等待 150-800ms 才能看到蛇头转动，低速时极为明显。

---

## 2. Solution

### 方案：立即视觉反馈（Immediate Visual Feedback）

不改变游戏循环或 tick 机制，仅修改渲染层和输入处理，让蛇头在按键按下时立即视觉朝新方向转动。

#### 变更 1：渲染层 — 用 `nextDirection` 绘制蛇头眼睛

**文件:** `public/src/render/room.js`
**行号:** 242
**当前代码:**
```javascript
if (state.direction.y === -1) { // up
```
**改为:**
```javascript
const displayDir = state.nextDirection || state.direction;
if (displayDir.y === -1) { // up
```

同时将所有后续 `state.direction` 引用替换为 `displayDir`（共 5 处，行 242-253）。

#### 变更 2：输入处理 — 方向变更后立即重绘

**文件:** `public/gameboy.html`
**行号:** 539-541
**当前代码:**
```javascript
if (dir) {
  e.preventDefault();
  state = changeDirection(state, dir);
}
```
**改为:**
```javascript
if (dir) {
  e.preventDefault();
  state = changeDirection(state, dir);
  render(ctx, state);
}
```

#### 完整替换映射

| 文件 | 行号 | 当前代码 | 改为 |
|------|------|---------|------|
| `public/src/render/room.js` | 239 | `if (state.direction.y === -1)` | `const displayDir = state.nextDirection \|\| state.direction;` + 后续用 `displayDir` |
| `public/src/render/room.js` | 242 | `state.direction.y === -1` | `displayDir.y === -1` |
| `public/src/render/room.js` | 245 | `state.direction.y === 1` | `displayDir.y === 1` |
| `public/src/render/room.js` | 248 | `state.direction.x === -1` | `displayDir.x === -1` |
| `public/src/render/room.js` | 253 | `state.direction` (else) | `displayDir` (else) |
| `public/gameboy.html` | 541 | 缺少 `render()` | 添加 `render(ctx, state);` |

## 3. Implementation Notes

### 需要修改的文件
- `public/src/render/room.js` — `drawSnake()` 函数，约 5 行
- `public/gameboy.html` — `keydown` 事件处理器，约 1 行

### 边界情况
1. **游戏暂停时 (`gameState === 'paused'`)**: `changeDirection` 返回原 state，不会触发视觉变化。安全。
2. **游戏结束时 (`gameover`/`won`)**: `changeDirection` 返回原 state。安全。
3. **Boss intro 状态**: `changeDirection` 有特殊处理（自动 dismiss），但会返回新 state。渲染后蛇头位置正确。安全。
4. **反向保护触发时**: `changeDirection` 返回 `{...state}`（shallow clone），`nextDirection` 保持不变。渲染仍然显示原方向，是正确行为。

### 风险
- **极小**。仅修改渲染方向和触发时机，不影响 state 逻辑
- 无 tick 循环改动，不影响碰撞检测、移动逻辑

### 作用域检查
- ✅ 不修改 `core.js` 的 `tick()` 或 `changeDirection()`
- ✅ 不修改游戏循环（`scheduleNextTick`, `runTick`）
- ✅ 不影响碰撞检测
- ✅ 不影响 E2E 测试（`__GAME_API__.setDirection` 直接操作 state，不经过渲染）

### 验收标准
1. 按下方向键 → 蛇头眼睛**立即**朝新方向转动，无需等待下一个 tick
2. 反向保护仍然生效（180 度掉头被阻止，眼睛不转动）
3. 蛇的实际移动节奏不变（tick 间隔不变）
4. 暂停/结束/BossIntro 状态下方向键不触发视觉变化

### 预计工作量
- 代码修改：≤ 10 分钟
- 测试验证：≤ 10 分钟
