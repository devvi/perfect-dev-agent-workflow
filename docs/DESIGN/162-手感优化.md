# DESIGN: 手感优化 — 方向键转蛇头即时视觉反馈

> **Issue:** #162 | **Depth:** light | **Type:** enhancement
> **PRD:** `docs/PRD/162-手感优化.md`

## Overview

Snake head direction changes currently only render visually **after the next tick** (150–800 ms delay), because the eye-drawing code reads `state.direction` instead of `state.nextDirection`. This design addresses input-to-visual lag by making two minimal changes:

1. **Render layer:** Use `state.nextDirection || state.direction` for snake head eyes
2. **Input handler:** Call `render()` immediately after `changeDirection()`

No changes to the game loop, tick timing, collision detection, or state logic.

## Data Flow

### Before (Current)

```
Player presses ← / → / ↑ / ↓
  ↓
changeDirection(state, dir) → state.nextDirection = dir (sync, OK)
  ↓
[Wait for next tick — up to 800ms]
  ↓
tick(): state.direction = state.nextDirection
  ↓
render() → drawSnake() reads state.direction → eyes face dir
  ↓
[Visual confirmed — only after tick boundary]
```

### After (New)

```
Player presses ← / → / ↑ / ↓
  ↓
changeDirection(state, dir) → state.nextDirection = dir (sync, OK)
  ↓
render(ctx, state) ← IMMEDIATE call
  ↓
drawSnake() reads displayDir = state.nextDirection || state.direction
  ↓
[Visual confirmed — same frame, <16ms]
```

## Implementation

### Change 1: Room renderer — `drawSnake()` eye direction

**File:** `public/src/render/room.js`, line 242

Replace `state.direction` references in the eye-drawing block with a `displayDir` variable that prefers `nextDirection`:

```js
// Before (line 242):
if (state.direction.y === -1) { // up

// After:
const displayDir = state.nextDirection || state.direction;
if (displayDir.y === -1) { // up
```

Replace all 4 eye-orientation `state.direction` references (lines 242, 245, 248, 251):

| Line | Current | New |
|------|---------|-----|
| 242 | `state.direction.y === -1` | `displayDir.y === -1` |
| 245 | `state.direction.y === 1` | `displayDir.y === 1` |
| 248 | `state.direction.x === -1` | `displayDir.x === -1` |
| 251 | `else` (fallthrough, uses `state.direction` implicitly) | `else` (uses `displayDir` implicitly — already handled by the early variable) |

### Change 2: Gameboy HTML — immediate re-render on input

**File:** `public/gameboy.html`, line 539-541

Add a `render()` call after the direction is changed:

```js
// Before:
if (dir) {
  e.preventDefault();
  state = changeDirection(state, dir);
}

// After:
if (dir) {
  e.preventDefault();
  state = changeDirection(state, dir);
  render(ctx, state);
}
```

## Boundary Conditions & Edge Cases

| Condition | Handling |
|-----------|----------|
| **Reverse direction (180° flip)** | `changeDirection` returns `{...state}` (shallow clone, `nextDirection` unchanged). `displayDir = state.nextDirection || state.direction` still shows the previous direction — eyes don't spin around. Correct behavior. |
| **Game paused (`gameState === 'paused'`)** | `changeDirection` returns `state` as-is (paused guard at line 450). Eyes unchanged. Safe. |
| **Game over / won** | `changeDirection` returns `state` as-is. Eyes frozen. Safe. |
| **Boss intro** | `changeDirection` dismisses intro and returns new state with `direction = nextDirection = {0,1}`. `render()` shows eyes pointing down, which matches the new downward-facing spawn. Safe. |
| **`nextDirection` is null/undefined** | Guard: `displayDir = state.nextDirection || state.direction` falls through to `direction`. Safe on initial state or malformed data. |
| **Tick fires between input and render** | `tick()` sets `direction = nextDirection` first, then `render()` reads `nextDirection || direction` — always correct. Order doesn't matter. |
| **Multi-key spam in one frame** | All calls processed sequentially by JS event loop. Last key wins. Each `render()` call shows the latest `nextDirection`. Transient intermediate states are skipped (same as current behavior, just faster). |

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| `render()` called while canvas not ready (title screen) | Low | Low | Guard: `gameState !== 'playing'` returns early in `changeDirection`; `render()` still safe to call (clears and draws title). |
| Performance: extra `render()` per keystroke | Very Low | Very Low | `render()` is ~0.5ms for full frame. Even 60 keystrokes/sec is negligible. |
| Visual flash if `nextDirection` and `direction` diverge | Low | Low | This is the intended design — showing `nextDirection` immediately is the goal. At tick, both converge. |
